## **🌟 v3.3 核心升級 (Major Updates)**

### **1\. 🔬 導入多元回歸模型 (Multivariable Regression Model)**

捨棄傳統簡易比較法，採用 IPMVP Option C 標準，納入影響能耗的三大關鍵變數：

* $\\beta\_1$ **(CDD)**：冷房度日 (Cooling Degree Days)，排除氣溫變化對空調能耗的干擾。  
* $\\beta\_2$ **(Traffic)**：來客數流量，排除人流熱負荷與自動門開關的影響。  
* $\\beta\_3$ **(Hours)**：營業時數，排除營運時間調整的差異。

公式核心：

$$E\_{adj\\\_base} \= \\beta\_0 \+ \\beta\_1(CDD) \+ \\beta\_2(Traffic) \+ \\beta\_3(Operating\\\_Hours)$$

### **2\. 📝 基準線協議書生成 (Baseline Protocol Generator)**

* **合規文件**：內建符合 ESCO 產業標準的「基準線確認協議書」模板。  
* **自動填寫**：系統會將您設定的模型係數 ($\\beta\_0, \\beta\_1...$) 自動帶入協議書中。  
* **預覽與列印**：支援一鍵預覽並列印 PDF，直接作為合約附件使用。

### **3\. 🛡️ 企業級風險控管**

* **非尋常調整 (Non-routine Adjustments)**：在協議書中明訂「裝修」、「設備增減」等排除條款，保障雙方權益。  
* **R² 檢定提示**：強調模型需符合統計顯著性 ($R^2 \\ge 0.85$)，提升數據公信力。

## **📖 操作指南 (v3.3 新增功能)**

1. **設定回歸係數**:  
   * 在 **「基準線模型 (M\&V)」** 分頁中，您會看到新的輸入面板。  
   * 輸入由歷史帳單分析得出的 $\\beta$ 係數（基礎負載、溫度係數等）。  
   * 調整當月的變動參數（如本月 CDD、來客數），觀察「調整後基準線」的變化。  
2. **生成協議書**:  
   * 點擊 M\&V 分頁右上角的 **「預覽協議書」** 按鈕。  
   * 系統會彈出正式的協議書視窗，確認內容無誤後可直接列印。
